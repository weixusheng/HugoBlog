gitignore文件的使用

git rm-r --cached .

git add .

git commit -m 'update .gitignore'

规则

常用的规则

1
2
3

/mtk/       # 过滤整个文件夹
*.zip       # 过滤所有.zip文件
/mtk/do.c   # 过滤某个具体文件

指定要将哪些文件添加到版本管理中

1
2
3

!src/        # 不过滤该文件夹
!*.zip       # 不过滤所有.zip文件
!/mtk/do.c   # 不过滤该文件

配置语法

以斜杠/开头表示目录； 以星号*通配多个字符； 以问号?通配单个字符 以方括号[]包含单个字符的匹配列表； 以叹号!表示不忽略(跟踪)匹配到的文件或目录；

此外，git 对于 .ignore 配置文件是按行从上到下进行规则匹配的，意味着如果前面的规则匹配的范围更大，则后面的规则将不会生效；
